- feat – a new feature is introduced with the changes; (new feature for the user, not a new feature for build script)
- fix – (bug fix for the user, not a fix to a build script)
- refactor: (refactoring production code, eg. renaming a variable; refactored code that neither fixes a bug nor adds a feature)
- chore – changes that do not relate to a fix or feature and don't modify src
  or test files (for example updating dependencies; updating grunt tasks etc; no production code change)
- docs – updates to documentation such as a the README or other markdown files
- style – (formatting, missing semi colons, etc; no production code change)
- test – (adding missing tests, refactoring tests; no production code change)

- perf – performance improvements
- ci – continuous integration related
- build – changes that affect the build system or external dependencies
- revert – reverts a previous commit
